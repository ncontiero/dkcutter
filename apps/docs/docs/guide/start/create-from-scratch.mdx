# Create a Template From Scratch

import { Steps } from "@/components/Steps";
import { PackageManagerTabs } from "@/components/PackageManagerTabs";

This guide will walk you through creating a simple `dkcutter` template from scratch. We'll build a template named `dkcutter-website-simple` for generating a basic HTML website.

This is a great way to understand the fundamentals of how `dkcutter` templates are structured.

## Let's get started!

<div className="steps">
<Steps>

### Create the Template Directory

First, create a directory for your new template and navigate into it.

```bash
mkdir dkcutter-website-simple
cd dkcutter-website-simple/
```

### Create the `dkcutter.json` file

This file is the heart of your template. It defines the variables that the user will be prompted for. Create a file named `dkcutter.json` with the following content:

```json title="dkcutter.json"
{
  "$schema": "https://dkcutter.ncontiero.com/schema.json",
  "projectName": "My Simple Website",
  "projectSlug": "{{ dkcutter.projectName|lower|replace(' ', '-')|trim }}"
}
```

**Explanation:**

-   `"$schema"`: (Optional) Specifies the schema for editor autocompletion and validation.
-   `"projectName"`: A variable that the user will be prompted for. We've given it a default value of `"My Simple Website"`.
-   `"projectSlug"`: A variable that is *derived* from `projectName`. It uses a template that converts the project name to lowercase, replaces spaces with hyphens, and removes any leading or trailing whitespace.

### Create the Template Files

Now, let's create the files that will be part of the generated project. Create the following directory structure:

```bash
mkdir -p template/{{dkcutter.projectSlug}}
```

This command creates a `template` directory, and inside it, a directory with a dynamic name `{{dkcutter.projectSlug}}`.

Inside that directory, create an `index.html` file:

```html title="template/{{dkcutter.projectSlug}}/index.html"
<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8">
    <title>{{ dkcutter.projectName }}</title>
  </head>
  <body>
    <h1>{{ dkcutter.projectName }}</h1>
  </body>
</html>
```

Notice the `{{ dkcutter.projectName }}` placeholders. DKCutter will replace these with the value of the `projectName` variable.

### Run DKCutter

Your template is now ready! Let's test it. From inside the `dkcutter-website-simple` directory, run `dkcutter` pointing to the current directory (`.`):

<PackageManagerTabs command="dkcutter@latest ." isExecuted />

DKCutter will prompt you for the `projectName`. Let's enter a new name:

```bash
✔ Project name? … My Awesome Site
✔ Project slug? … my-awesome-site
✔ Project created!
```

### Check the Output

DKCutter will generate a new folder in your current directory named `my-awesome-site`. If you inspect the generated `index.html`, you'll see that the placeholders have been replaced with your project name.

Congratulations! You've just created and used your first `dkcutter` template.

</Steps>

## Alternative: Using `--init`

Instead of creating a template entirely from scratch, you can use the `--init` flag to generate a boilerplate template structure. This is a great starting point.
